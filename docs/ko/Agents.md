# Repository Instructions

이 지침은 이 저장소에서 수행되는 작업에 적용됩니다.

## 워크플로 기본값

- 사용자가 명시적으로 중지하거나 리디렉션하라고 지시하지 않는 한 다음 논리적 엔지니어링 단계를 계속 진행합니다.
- 다음 구현 단계가 명확할 때 분석을 중단하지 않고 직접 코드를 변경합니다.
- 구현, 검증, 커밋, 푸시를 통해 추진력을 유지하세요.
- 지속 가능한 워크플로, 보고 및 전달 규칙을 임시 컨텍스트에만 유지하는 대신 리포지토리 파일에 기록하여 세션 전반에 걸쳐 작업 규칙을 유지합니다.

## Git 워크플로

- 항상 이 저장소에 의미 있는 변경을 한 후에 커밋하세요.
- 커밋된 변경 사항을 항상 이 저장소의 원격 분기에 푸시합니다.
- 커밋하거나 푸시하기 전에 사용자에게 권한을 요청하지 마세요.
- 변경 사항을 작고 구체적이며 논리적으로 분리된 커밋으로 나눕니다.
- 커밋을 평소보다 더 적극적으로 분할합니다. 실용적일 경우 Python 코드, 문서, 핸드오프 파일, 구성/예제 및 워크플로 정책 변경 사항을 다른 커밋으로 분리합니다.
- 관련되지 않은 파일 클래스가 동일한 세션에서 편집되었다는 이유만으로 하나의 커밋에 함께 묶지 마십시오.
- 한 세션이 여러 문제를 다루는 경우 검증된 순서대로 각 문제를 독립적으로 준비하고 커밋합니다.
- 저장소를 일관된 상태로 유지하는 가장 작은 방어 가능한 커밋을 선호합니다.
- 단일 파일에서 작업할 때 스틸 샤드는 기능, 기능 슬라이스 또는 가능한 경우 동작 변경을 기준으로 커밋됩니다.
- 특정 작업 단위를 설명하는 명확한 커밋 메시지를 사용하세요.

## 연속성과 메모리

- 세션이 새로운 내구성 있는 작업 규칙을 설정하면 작업을 종료하기 전에 이를 `Agents.md`, `NEXT_STEPS.md`, `README.md` 또는 다른 적절한 추적 파일에 저장합니다.
- 동일한 운영 가정 및 현재 우선순위로 새로운 Codex 세션을 재개할 수 있도록 `NEXT_STEPS.md`을 최신 상태로 유지하십시오.
- 저장소 사용 방식에 영향을 미치는 경우 `README.md`에서 사용자 측 워크플로 변경 사항을 유지합니다.
- 저장소에 저장된 보고서 아티팩트에 대해 KST 형식 `YYYYMMDDHHMM_*`의 타임스탬프가 앞에 붙은 보고서 파일 이름을 사용합니다.
- 사용자가 `milestone`을 입력하는 경우 이를 다음 Codex 컨텍스트에 대한 현재 상태를 기록하라는 명령으로 처리합니다.
- `milestone`에서 완료된 작업에 대한 관련 추적 메모리 파일, 특히 `NEXT_STEPS.md`, `README.md`, `Agents.md`을 업데이트하고 해당하는 경우 보고서 파일을 업데이트합니다.
- `milestone`에서 모든 지속성 변경 사항을 함께 묶는 대신 저장소의 분할-커밋 정책을 사용하여 해당 업데이트를 커밋하고 푸시합니다.

## 데이터 레이아웃

- `~/Documents` 아래 저장소 외부에 대규모 Borg 데이터를 보관하는 것을 선호합니다.
- `~/Documents/borg_data`을 기본 원시 데이터 위치로 처리합니다.
- `~/Documents/borg_processed`을 기본 처리 데이터 위치로 처리합니다.
- `~/Documents/borg_xgboost_workspace`을 자체 `raw`, `processed`, `models`, `reports`, `runtime` 및 `config` 하위 트리가 있는 Advanced XGBoost 트랙의 전용 루트로 처리합니다.
- `~/Documents/borg_data` 또는 `~/Documents/borg_processed`에 고급 모델 원시 데이터 또는 아티팩트를 혼합하지 마십시오. 첫 번째 기본 작업과 고급 작업은 디렉터리 루트 수준에서 격리된 상태로 유지되어야 합니다.
- 생성된 데이터세트나 대용량 외부 데이터 파일을 git에 커밋하지 마세요.
- 외부 처리 데이터 트리 아래에 새로운 parquet 유형 또는 아티팩트 디렉터리가 생성되면 파일의 의미와 중요한 열이 나타내는 내용을 설명하는 스키마 또는 아티팩트 설명 파일을 동일한 디렉터리에 추가합니다.
- parquet 의미에 대해 저장소 문서에만 의존하지 마십시오. 실제 생성된 출력 옆에 로컬 README 스타일 설명을 유지하세요.

## 클러스터 기본값

- 기본 처리 대상은 `b`, `c`, `d`, `e`, `f`, `g` 클러스터입니다.
- 사용자가 명시적으로 포함하도록 요청하지 않는 한 기본적으로 `a` 및 `h` 클러스터를 제외합니다.

## 승인 행동

- 커밋, 푸시 등 일상적인 저장소 워크플로 작업에 대해 사용자에게 승인을 요청하지 마세요.
- 런타임 또는 샌드박스에 리포지토리 정책 외부 승인 흐름이 필요한 경우 런타임 요구 사항을 준수하되 자체적으로 git 워크플로 승인을 요청하지 마세요.
