# Forecaster 평가 및 해석

날짜: 2026-03-19

## horizon

이 보고서는 평가된 예측 기준 Result와 Borg 추적 실험 중에 생성된 해석 Result를 기록합니다.

## 기본 기준 프로필

기본 프로필: `base`

검증 Result:

- 유효성 검사 행: `4,810,777`
- 검증 긍정: `1,448`
- 검증 긍정률 : `0.0301%`
- 평균 정밀도: `0.0074076`
- 정밀도@0.1%: `0.0122661`
- 회상@0.1%: `0.0407459`
- 정밀도@1%: `0.0070676`
- 리콜@1%: `0.2348066`

이 계산을 읽는 방법:

- 검증 positive률(`0.0301%`)은 검증 분할의 기본 positive률이며 `1,448 / 4,810,777`으로 계산됩니다.
- `Precision@0.1%`은(는) 모든 검증 행을 `risk_score` 내림차순으로 정렬하고, 행 중 가장 위험도가 높은 `0.1%`을 선택하고, 해당 조각 내에서 참positive의 비율을 계산한다는 의미입니다.
- `4,810,777` 유효성 검사 행을 사용하면 상단 `0.1%` 슬라이스에 약 `4,810` 행이 포함됩니다.
- `Precision@1%`은 유효성 검사 행 중 가장 위험도가 높은 `1%`에 대해 동일한 계산을 의미하며, 약 `48,107` 행입니다.
- 정밀 리프트 해석은 해당 정밀도 값을 기본 비율과 비교합니다.
- `base` 프로필 `1.2266% / 0.0301% ≈ 40.7`의 경우 상위 `0.1%` 순위 조각은 무작위 선택보다 positive에서 약 `40x` 밀도가 더 높습니다.
- `base` 프로필 `0.7068% / 0.0301% ≈ 23.5`의 경우 상위 `1%` 순위 슬라이스는 무작위 선택보다 positive에서 약 `23x` 밀도가 더 높습니다.

ML 순위와 ML이 없는 무작위 선택의 즉각적인 효과:

| 미터법 | ML 없음 기준(무작위/기본 비율) | ML 기준선(`base`) | 효과 |
| --- | ---: | ---: | ---: |
| 전체 검증 세트의 긍정적인 비율 | `0.0301%` | `0.0301%` | 동일한 기본 데이터 |
| 상단 `0.1%` 슬라이스의 정밀도 | `0.0301%` | `1.2266%` | 약 `40.7x` 이상 |
| 상단 `1%` 슬라이스의 정밀도 | `0.0301%` | `0.7068%` | 약 `23.5x` 이상 |
| 상단 `0.1%` 슬라이스 리콜 | `0.1%` 무작위 순위 예상 | `4.0746%` | 약 `40.7x` 이상 |
| 상단 `1%` 슬라이스 리콜 | `1%` 무작위 순위 예상 | `23.4807%` | 약 `23.5x` 이상 |

표 해석:

- ML 없는 기준선은 무작위 선택이며, 순위 신호가 없기 때문에 예상 정밀도가 기본 비율과 같습니다.
- 무작위 순위에서 행의 상위 `0.1%` 또는 `1%`을 선택하면 평균적으로 모든 긍정의 약 `0.1%` 또는 `1%`이 복구됩니다.
- ML 모델은 검사된 슬라이스의 양의 밀도와 해당 슬라이스에서 복구된 모든 오류의 비율을 모두 크게 증가시킵니다.

위험 점수 계산 방법:

- 각 입력 기능은 훈련 분할 통계를 사용하여 표준화됩니다. null 값은 훈련 중앙값으로 채워진 다음 각 값은 `(value - mean) / std`으로 변환됩니다.
- 특성 가중치는 훈련 데이터에서 양수 행의 평균 표준화 특성 값과 음수 행의 평균 표준화 특성 값 간의 차이로 계산됩니다.
- 최종 `risk_score`은 선택한 기능 세트 전체에 걸쳐 표준화된 기능 값의 선형 가중치 합계입니다.
- `risk_score`이 높을수록 행이 이 기준 순위 모델에서 포지티브 클래스와 더 유사하다는 의미입니다.
- 점수는 보정된 확률이 아닌 순위 값으로 사용됩니다.

재현율 계산 방법:

- `Recall@0.1%` 의미: `risk_score` 내림차순으로 정렬한 후 행의 상위 `0.1%`을 가져와 해당 슬라이스 내에 포함된 모든 검증 긍정의 비율을 측정합니다.
- `Recall@1%`은 상위 `1%` 순위 슬라이스에서도 동일한 계산을 의미합니다.
- `base` 프로필의 경우 `Recall@0.1% = 0.0407459`은 상위 `0.1%` 가장 높은 위험 행에 모든 참positive의 약 `4.07%`이 포함되어 있음을 의미합니다.
- `base` 프로필의 경우 `Recall@1% = 0.2348066`은 상위 `1%` 가장 위험도가 높은 행에 모든 참positive의 약 `23.48%`이 포함되어 있음을 의미합니다.
- 재현율은 정밀도와는 다른 질문에 답합니다. 정밀도는 "선택한 행 중 몇 개가 실제로 긍정적입니까?"를 묻고, 재현율은 "모든 참positive 중에서 성공적으로 복구한 행 수는 몇 개입니까?"를 묻습니다.

## 대체 롤링 프로필

대체 프로필: `base_plus_roll`

검증 Result:

- 유효성 검사 행: `4,810,777`
- 검증 긍정: `1,448`
- 검증 긍정률 : `0.0301%`
- 평균 정밀도: `0.0071694`
- 정밀도@0.1%: `0.0222453`
- 회상@0.1%: `0.0738950`
- 정밀도@1%: `0.0069013`
- 리콜@1%: `0.2292818`

해석:

- `base`은 전체 순위 품질에 대한 최고의 기본 프로필로 유지되었습니다.
- `base_plus_roll`은 최고 위험 경고 슬라이스를 개선했습니다.
- 전체 시간적 특징 확장은 전체 평균 정밀도를 감소시켰으며 기본값으로 유지되지 않았습니다.

## 클러스터 수준 해석

클러스터별 동작이 고르지 않았으므로 집계 점수 뒤에 숨겨서는 안 됩니다.

- 클러스터 `b` : 순위 성적이 비교적 강함
- 클러스터 `c` : 순위 성적이 비교적 강함
- 클러스터 `d`: 혼합 성능
- 클러스터 `e`: 평가된 분할에 긍정적인 검증 예가 없습니다.
- 클러스터 `f`: 성능 저하
- 클러스터 `g`: 성능 저하

해석:

- 기준선은 클러스터 전체에 균일하게 일반화되지 않습니다.
- 향후 비교에는 클러스터별 측정항목이 계속 포함되어야 합니다.
- 집계된 지표만으로는 모델 품질을 결정하는 데 충분하지 않습니다.

## 기능 해석

기준선에서 가장 강한 특징 가중치는 스케줄링 및 리소스 사용 신호에 집중되었습니다.

내보낸 기능 순위에서 관찰된 최고 순위 신호:

- `scheduling_class`
- `avg_mem`
- `max_mem`
- `avg_mem_utilization`
- `max_mem_utilization`
- `max_cpu_utilization`
- `max_cpu`
- `avg_cpu_utilization`
- `avg_cpu`

해석:

- 메모리 수준과 메모리 활용도 신호는 기준선 분리에서 특히 중요했습니다.
- CPU 활용도와 CPU 레벨 신호도 중요했습니다.
- `req_cpu`, `req_mem`, 특히 `event_count`은 이 기준선에서 직접적인 자원 사용 신호보다 약했습니다.

## 주요 결론

- 현재 `base` 프로필이 가장 강력한 기본 기준입니다.
- `base_plus_roll` 프로필은 최상위 경고 분류 시나리오에 유용합니다.
- 다음 개선은 현재 선형 위험 점수 기준에 필터링되지 않은 시간적 기능을 더 추가하는 것이 아니라 더 강력한 모델 클래스에서 나와야 합니다.
